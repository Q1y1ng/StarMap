### 标注格式

Ground Truth 采用结构化 Markdown，包含 5 个必需章节：
- **考试信息**：考试名称、科目、年级、总分
- **试卷内容**：逐题编号的试题文本
- **学生作答**：逐题编号的学生答案
- **成绩信息**：总得分
- **小分/错题汇总**：Markdown 表格，含题号、得分、满分、知识点字段

## 三、基线对照方案

为科学评估 StarMap 多页联合解析效果，设置三条基线：

### Baseline 1: PaddleOCR（传统 OCR）
- 方法：逐张图片进行本地 OCR 识别
- 特点：无语义理解，仅提取文字
- 预期：高文字召回但低结构化

### Baseline 2: SingleVision（单页 Vision）
- 方法：每页图片独立调用 Vision API
- 特点：逐页理解但丧失跨页关联
- 预期：单页内容准确但跨页对应错误

### Baseline 3: StarMap（多页联合 Vision）— 本系统方案
- 方法：全部图片一次性送入 Vision API
- 特点：模型同时看到所有页面，建立跨图片上下文
- 预期：整体结构化最优

## 四、实验流程

1. 准备 ExamBench-v1 数据集（Ground Truth）
2. 分别用三种方案处理全部样本
3. 计算各项评价指标
4. 对比分析，得出结论
